# ✅ AI 刷题「一页速查 Prompt 集」

> 适用：LeetCode / ACM / 牛客 / 洛谷 等在线评测
>
> 语言：Java 8
>
> 前提：已上传题目图（或已给出题面）

------

## ① 首次提问（强烈推荐）

```
请你作为一位拥有 10 年以上经验的资深 Java 算法工程师，结合我上传的 2 张图片，解决这道编程题。其中：
图1 是题目描述与约束条件
图2和图3 是输入与输出样例（包含 Case 1 与 Case 2）

请先复述题目核心逻辑、输入格式、输出格式以及关键约束条件，确保理解无误后，再开始编码。

基于题目要求用 Java 8 编写可直接在在线评测系统（OJ）提交的完整代码，采用标准 ACM 输入输出模式，要求：
1. 代码可直接运行（包含 main 方法）
2. 仅使用 java.io / java.util 标准库, 必须显式导入具体类

3. 输入输出格式与题目样例严格一致（包括空格、换行、大小写）
4. 必须通过题目给出的所有样例，尤其是 Case 1 与 Case 2，输出结果需完全一致
5. 覆盖常规输入、边界条件及极端场景
6. 时空复杂度最优
7. 代码精简、规范、注释清晰
8. 仅输出可在在线评测系统直接提交的完整 Java 代码，无任何多余文字
```

```
请你作为一位拥有 10 年以上经验的资深 Java 算法工程师，结合我上传的图片（图1 题目描述，图2 输入输出样例），解决这道编程题。

请先复述题目核心逻辑、输入格式、输出格式以及关键约束条件，确认理解无误后，再开始编码。

要求：
1. 使用 Java 8，标准 ACM 输入输出
2. 包含 main 方法，可直接在 OJ 提交
3. 仅使用 java.io / java.util
4. 输入输出格式与样例严格一致
5. 覆盖边界与极端情况
6. 时空复杂度最优
7. 注释清晰、代码精简

仅输出完整 Java 代码，无任何多余文字。
```

## ② WA（答案错误）—— 有具体数据

```
这是我上一轮提交的 Java 代码（见下方）。
它在在线评测系统中未通过，具体反馈如下：

- 输入用例：
【粘贴 Case 2 或失败的输入】

- 我的程序输出：
【粘贴你的程序实际输出】

- 期望输出：
【粘贴 OJ 给出的正确输出】

请你：
1. 分析两者差异，找出逻辑错误的根本原因
2. 指出是算法问题、边界条件问题，还是输入输出处理问题
3. 给出一份可直接替换提交的修正后完整 Java 代码
4. 保留 ACM 输入输出格式、Java 8、无第三方库

不要解释无关内容，直接给出可运行的修正代码。
```

```
这是我上一轮提交的 Java 代码（见下方）。
它在 OJ 中返回 Wrong Answer。

失败用例输入：
【粘贴输入】

我的程序输出：
【粘贴实际输出】

【粘贴正确答案】

请：
1. 对比差异，找出根本原因
2. 修复逻辑或边界错误
3. 输出一份可直接替换提交的完整 Java 代码
```

## ③ WA（答案错误）—— 无具体数据

```
我上一轮提交的 Java 代码未通过在线评测，OJ 只返回 Wrong Answer，没有给出具体测试用例。

已知信息：
- Case 1 可通过
- Case 2 失败（但我看不到输入）

请你：
1. 基于题目原始描述（见之前图片），推测 Case 2 最可能考察的边界场景
2. 检查我代码中是否存在：
   - 边界条件遗漏
   - 整数溢出
   - 输出格式错误（空格 / 换行）
   - 初始化问题
3. 给出一份修复后的完整 Java 代码
```

```
我上一轮提交的 Java 代码未通过 OJ，
只知道 Case 1 通过，Case 2 失败，但没有具体测试用例。

请：
1. 基于原题描述，推测 Case 2 最可能的边界场景
2. 检查是否存在：
   - 边界条件错误
   - 输出格式问题
   - 初始化 / 溢出问题
3. 给出一份修复后的完整 Java 代码
```

## ④ TLE / MLE（超时 / 超内存）

```
我的 Java 代码在 OJ 上报 TLE（或 MLE）。

已知数据规模：
- n 最大为 【填写，如 200000】
- 当前时间复杂度约为 【如 O(n²)】

请：
1. 分析性能瓶颈
2. 给出更优算法思路
3. 输出一份时间复杂度最优的完整 Java 代码
```

## ⑤ CE（编译错误）

```

错误信息：
【粘贴 CE 信息】

请：
1. 指出不兼容原因（Java 版本 / import / 语法）
2. 给出一份完全符合 Java 8 + OJ 环境的修正代码
```

```

【简述题型，如：区间 DP / 贪心 / BFS】

失败现象：
【如：Case 2 WA / TLE / 输出多一个空格】

请：
1. 分析最可能出错的原因
2. 给出修改建议
3. 输出一份可直接提交的完整 Java 代码
```

## ✅ 使用小贴士（非常重要）

- ✅ **图片必须清晰**（尤其是输入格式）
- ✅ 第一次一定让 AI **先复述题意**
- ✅ 第二次提问**一定要给“差异”**
- ✅ 永远不要只说「帮我改一下」

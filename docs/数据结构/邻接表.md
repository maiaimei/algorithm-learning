# 邻接表（Adjacency List）

## 一、定义

邻接表是一种专门用于表示 **图** 或 **树**（树是特殊的图）的结构，核心作用是记录「每个节点与哪些节点直接相连」。

形象理解：把每个节点看作一个人，邻接表就是每个人的「通讯录」，只记录和自己直接来往（直接相邻）的人。

## 二、特点

- 简洁高效：只存储节点的直接相邻关系，不浪费空间
- 用途广泛：树/图的遍历（BFS、DFS）、找路径、找叶子节点等所有树/图相关算法，都依赖邻接表
- 易实现：
    - 在Python中常用字典（或默认字典）\+ 列表实现
    - 在Java中常用 `List<List<Integer>>` = 「大列表套小列表」实现
        - 外层列表：代表所有节点
        - 内层列表：代表当前节点的所有邻居

## 三、实例演示

### 1\. 示例树结构

输入：

```
4  
-1 2 3 2  
1 2  
1 3  
3 4

  1(-1)
 /   \
2(2) 3(3)
      /
    4(2)
```

解释：

- 输入的第一行是一个整数 \( N \)，表示树中节点的数量。
- 第二行包含 \( N \) 个用空格分隔的整数，表示树中每个节点的值。
- 接下来的 \( N-1 \) 行，每行包含两个用空格分隔的整数 \( a \) 和 \( b \)，表示树中一条边的**起点节点**和**终点节点**的编号。

连线是：

1-2，1-3，3-4

### 2\. 对应的邻接表

用「节点: 相邻节点列表」的形式表示：

```
1: [2, 3]   → 节点1 连着 2 和 3
2: [1]      → 节点2 只连着 1
3: [1, 4]   → 节点3 连着 1 和 4
4: [3]      → 节点4 只连着 3
```

## 四、Python实现方式

### 1\. 基础实现（字典\+列表）

```python
# 直接定义完整邻接表
adj = {
    1: [2, 3],
    2: [1],
    3: [1, 4],
    4: [3]
}
    
```

### 2\. 代码中常用方式（先初始化空表，再填边）

对应之前树的代码，先创建空邻接表，后续通过输入的边信息填充：

```python
from collections import defaultdict

# 初始化一个空的邻接表（defaultdict自动为新节点创建空列表）
adj = defaultdict(list)

# 后续填充边（示例：添加边1-2、1-3、3-4）
edges = [(1,2), (1,3), (3,4)]
for a, b in edges:
    adj[a].append(b)  # 节点a添加邻居b
    adj[b].append(a)  # 节点b添加邻居a（树是无向图，双向添加）
```

## 五、Java实现方式

### 1\. 初始化空邻接表

```java
// 节点编号从 1 ~ n
int n = 4;  // 节点数量
List<List<Integer>> adj = new ArrayList<>();

// 给每个节点初始化一个空的邻居列表
for (int i = 0; i <= n; i++) {
    adj.add(new ArrayList<>());
}
```

### 2\. 添加边（树是无向图，双向添加）

```java
// 添加边 a <-> b
adj.get(a).add(b);
adj.get(b).add(a);
```

### 3\. 一句话记忆

**Java 邻接表 = List<List\<Integer>>**

- 外层：所有节点
- 内层：当前节点的邻居
- 无向树 / 图：添加边必须**双向添加**

## 六、核心用途

邻接表的核心价值是「提供节点的相邻关系」，支撑所有树/图操作：

- 遍历树/图：通过邻接表找到当前节点的下一个可访问节点

- 找路径：比如之前代码中，通过邻接表的邻居关系，用BFS找到两个叶子节点之间的路径

- 找节点属性：比如判断叶子节点（邻接表中，非根节点且邻居数为1的节点）

## 七、一句话总结

邻接表 = 每个节点的「邻居清单」，记录节点的直接相邻关系，是树/图算法的基础工具。

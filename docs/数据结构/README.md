# 数据结构（Data Structure）

## 一、定义与作用

**数据结构**是计算机中**组织和存储数据的方式**，用于高效地支持：访问、插入、删除、查找、修改

✅ **数据结构 = 数据的组织形式 + 支持的操作**

同样的数据，采用不同的组织结构，会导致：

- **时间效率不同**（操作快慢：O(1)、O(n)、O(log n)）
- **空间效率不同**（内存占用）

------

## 二、两大核心层面

```
数据结构
├── 逻辑结构（数据之间的关系）
└── 物理 / 存储结构（内存中的实际存储方式）
```

✅ **同一逻辑结构，可以用不同物理结构实现**

------

## 三、逻辑结构（Logical Structure）

描述数据元素之间的**逻辑关系**，与计算机如何存储无关。

| 逻辑结构      | 元素关系       | 特点                   | 示例                  |
|-----------|------------|----------------------|---------------------|
| 集合结构      | 无关系        | 仅同属一个集合              | 哈希表、集合              |
| 线性结构      | 1 : 1（一对一） | 除首尾外，每个元素仅有一个前驱和一个后继 | 数组、链表、单链表、双向链表、栈、队列 |
| 树形结构      | 1 : n（一对多） | 非线性结构，有根、分层、无环       | 二叉树、二叉搜索树、堆         |
| 图状 / 网状结构 | m : n（多对多） | 非线性结构，可含环、可有向或无向     | 有向图、无向图             |

------

## 四、物理结构 / 存储结构（Physical / Storage Structure）

描述数据在计算机内存中的**实际存储方式**。

| **存储方式** | **特点** | **优点**     | **缺点** |
|:---------|:-------|:-----------|:-------|
| 顺序存储     | 连续内存   | 随机访问快 O(1) | 插入删除慢  |
| 链式存储     | 指针连接   | 插入删除快      | 不能随机访问 |
| 索引存储     | 索引表    | 查找快        | 额外空间   |
| 散列存储     | 哈希函数   | 查找平均 O(1)  | 冲突处理   |

------

## 五、逻辑结构 vs 物理结构

| 对比项     | 逻辑结构 | 物理结构 |
|---------|------|------|
| 关注点     | 数据关系 | 内存存放 |
| 是否依赖计算机 | ❌ 否  | ✅ 是  |

------

## 六、数据结构 vs 算法

| 概念   | 作用       |
|------|----------|
| 数据结构 | “容器”：如何存 |
| 算法   | “方法”：如何用 |

✅ 数据结构 + 算法 = 高效程序

------

## 七、一句话总结

逻辑结构 = 数据之间是什么关系

物理结构 = 数据在内存里怎么摆放

# 线性表

## 一、线性表的概念

**线性表（Linear List）** 是由 **n（n ≥ 0）个相同类型的数据元素** 组成的一个**有限序列**。

形式化表示：

```
L=(a₁,a₂,…,a_n)
```

其中：

- n为线性表的长度

- a₁是第一个元素（无前驱）

- a_n是最后一个元素（无后继）

- 除首尾外，每个元素都有且仅有一个**直接前驱**和一个**直接后继**

------

## 二、线性表的特点

| 特点          | 说明                   |
|-------------|----------------------|
| **元素个数有限**  | 线性表是有限序列             |
| **元素类型相同**  | 所有元素属于同一数据类型         |
| **元素具有顺序性** | 元素按先后次序排列            |
| **一对一关系**   | 除首尾外，每个元素只有一个前驱和一个后继 |
| **逻辑结构简单**  | 是最基本、最常用的数据结构之一      |

线性表是**逻辑结构**，存储方式可以是顺序存储或链式存储。

------

## 三、常见线性表的分类

### 按存储结构划分

#### 顺序表（Sequence List）

**存储方式**：用一段**连续的存储单元**依次存放元素

**代表实现**：数组（Array）

✅ 优点：

- 随机访问快（O(1)）

- 存储密度高

❌ 缺点：

- 插入、删除需要移动大量元素

- 容量固定（静态顺序表）

📌 典型操作：按位置访问、遍历

------

#### 链表（Linked List）

逻辑相邻的数据元素在物理位置上可能不相邻。在链表中，数据元素之间的逻辑关系是由结点中的指针指示的。

结点示例：<img src="./images/LinkedListElement.png" style="zoom: 33%;" />

**存储方式**：通过**指针/引用**将零散的内存块连接

常见链表类型：

- 单链表：每个结点含数据 + 指向下一个结点的指针

- 循环链表：尾结点指向头结点

  <img src="./images/CircularLinkedList.png"  />

- 双向链表：结点含前驱指针和后继指针

- 静态链表：用数组模拟链表（无指针语言中使用）

✅ 优点：

- 插入、删除效率高（O(1)，已知位置）

- 动态扩容

❌ 缺点：

- 不支持随机访问

- 存储开销大（指针）

------

### 按功能和使用方式划分

| 线性表           | 特点                   |
|---------------|----------------------|
| **栈（Stack）**  | 先进后出（LIFO），只允许在一端操作  |
| **队列（Queue）** | 先进先出（FIFO），一端入队，一端出队 |
| **串（String）** | 字符组成的线性表             |
| **广义表**       | 元素可以是原子或子表（非线性扩展）    |

------

## 四、常见线性表对比总结

| 线性表 | 存储结构  | 插入/删除   | 访问      | 应用场景     |
|-----|-------|---------|---------|----------|
| 顺序表 | 连续存储  | 慢（O(n)） | 快（O(1)） | 查找频繁     |
| 单链表 | 非连续   | 快（O(1)） | 慢（O(n)） | 插入删除频繁   |
| 栈   | 顺序/链式 | 仅限一端    | 受限      | 表达式求值、递归 |
| 队列  | 顺序/链式 | 两端操作    | 受限      | 任务调度、缓冲区 |

------

## 五、一句话总结

**线性表是具有相同数据类型的 n 个元素的有限序列，元素之间是一对一的关系；常见线性表包括顺序表、链表、栈、队列、串等。**
